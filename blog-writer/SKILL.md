# Blog Writer

Create high-quality blog posts through a structured research, outline, and writing workflow.

## Workflow Overview

Blog writing follows this sequence:

1. **Research Phase** - Gather sources, verify facts, analyze competitors
2. **Outline Phase** - Structure content with clear sections and key points
3. **Writing Phase** - Draft full article following style guidelines
4. **Review Phase** - Check against quality standards and optimize for SEO

## Phase 1: Research

Before writing, conduct thorough research using the checklist at `/home/ubuntu/skills/blog-writer/references/research_checklist.md`.

**Key activities**:
- Define core question and audience knowledge level
- Gather 3-5 primary sources (academic papers, industry reports, expert quotes)
- Review 3-5 competitor articles to identify gaps
- Collect statistics, examples, and case studies
- Verify all facts by tracing to original sources

**Output**: Research notes with verified sources and key findings.

## Phase 2: Outline

Create a structured outline using the template at `/home/ubuntu/skills/blog-writer/templates/outline_template.md`.

**Outline structure**:
- Working title with target keyword
- Hook/opening (compelling question or scenario)
- 3-5 main sections with descriptive H2 headings
- Key points and supporting details for each section
- Conclusion with takeaways and call-to-action
- Meta information (title, description, links)

**Best practices**:
- Use descriptive headings (not "Introduction" or "Section 1")
- Limit each section to one main idea
- Include specific examples or data points
- Plan for visual elements (screenshots, diagrams)

**Output**: Detailed outline with all sections mapped out.

## Phase 3: Writing

Draft the full article following the style guide at `/home/ubuntu/skills/blog-writer/references/style_guide.md`.

**Writing guidelines**:

**Tone**: Conversational yet professional, using active voice and personal pronouns.

**Structure**: Short paragraphs (3-4 sentences), subheadings every 200-300 words, bullet points for lists.

**Opening**: Start with a compelling hook - question, statistic, or relatable scenario. Avoid generic introductions.

**Body**: Each section should have a clear H2 heading, supporting examples, and smooth transitions between ideas.

**Conclusion**: Summarize 3-5 key takeaways and end with a clear call-to-action.

**Length targets**:
- Short-form: 600-800 words
- Standard: 1,200-1,500 words
- Long-form: 2,000-3,000 words

## Phase 4: Review

Check the draft against quality standards:

**Content quality**:
- Does the opening hook grab attention?
- Are claims supported by credible sources?
- Are examples concrete and relevant?
- Does the conclusion provide clear takeaways?

**Readability**:
- Are paragraphs short (3-4 sentences)?
- Are headings descriptive and scannable?
- Is technical jargon defined on first use?
- Are transitions smooth between sections?

**SEO optimization**:
- Is target keyword in H1, first paragraph, and one H2?
- Is meta description 150-160 characters?
- Are 2-3 internal links included?
- Do images have descriptive alt text?

## Common Patterns

**How-to guide**: Problem → Solution steps → Examples → Conclusion

**Listicle**: Hook → Item 1 (with explanation) → Item 2 → ... → Summary

**Case study**: Challenge → Approach → Results → Lessons learned

**Comparison**: Context → Option A analysis → Option B analysis → Recommendation
